# 数据库类型
- ### 关系型数据库
  - #### 概念
    - ##### 关系型数据库模型是将复杂的数据结构用较为简单的二元关系(二维表)来表示, 我们可以用结构化查询语言(SQL)对数据库进行操作, 具有代表性质的数据库管理系统有: Oracle, MySQL, SQL Server, DB2
  - #### 类型
    - ##### Oracle
      - ###### 世界上最流行的数据库, 收费高
    - ##### DB2
      - ###### 适合大型的分布式应用系统, 繁琐
    - ##### SQL Server
      - ###### 只能在Windows下运行, 收费高
    - ##### MySQL
      - ###### 开放源代码
      - ###### 跨平台
      - ###### 轻量级
      - ###### 成本低
- ### 非关系数据库
  - #### NOSQL泛指非关系型数据库, 关系型数据库在超大规模和高并发的web2.0纯动态网络已经暴露很多问题, NOSQL数据库的产生就是为了解决大规模数据集合多重数据种类带来的挑战, 尤其是大数据应用难题, 常见的非关系型数据库管理系统有: Memcached, MongoDB, redis, HBase等

# 数据库表 
- ### 二维表
  - #### 二维表是由行和列组成的, 行被称为记录, 列被称为这些记录的字段
- ### 创建表 (DDL)
```sql
## 创建一个学生表

CREATE table t_student(
	sid INT(6), 
	sname VARCHAR(10),
	sex CHAR(5),
	age INT(3),
	enterdate date,
	classname VARCHAR(10),
	email VARCHAR(15)
);

## 查看表的结构: 展示表的字段的详细信息

DESC t_student;

## 查看表中的数据

SELECT * FROM t_student;

## 查看建表语句

SHOW CREATE table t_student;

## 添加一张完全一模一样的表

CREATE TABLE t_student2 AS SELECT * FROM t_student;

## 添加一个表, 但是无数据

CREATE TABLE t_student3 AS SELECT * FROM t_student WHERE 1 = 2;

## 添加一个表, 只要部分列, 部分数据

CREATE TABLE t_student4 AS SELECT sid, sname, age FROM t_student WHERE sid = 2;

## 删除表中数据

DELETE FROM t_student; -- 数据操作语言, 为DML操作

TRUNCATE FROM t_student; -- 数据定义语言, 为DDL操作
```

- ### 单表查询知识点总结
```sql
## select 语句总结
SELECT column, group_function(column) FROM table 
[WHERE condition]
[GROUP BY group_by_expression]
[HAVING group_condition]
[ORDER BY column];
## 注意顺序固定, 不可以改变顺序

## select语句的执行顺序 
FROM - WHERE - GROUP BY - SELECT - HAVING - ORDER BY

## 单表查询练习
## 列出工资最小值小于2000的职位
SELECT job, min(sal) FROM emp GROUP BY job HAVING min(sal) < 2000;

## 列出平均工资大于1200的部门和工作的搭配组合
SELECT deptno, job, AVG(sal) FROM emp GROUP BY deptno, job HAVING AVG(sal) > 1200 ORDER BY deptno;

## 统计[人数小于4的]部门的平均工资
SELECT deptno 部门, COUNT(1) 总人数, AVG(sal) 平均工资 FROM emp GROUP BY deptno HAVING COUNT(1) < 4;

## 统计各部门的最高工资, 排除最高工资小于3000剩下的部门
SELECT deptno, max(sal) FROM emp GROUP BY deptno HAVING max(sal) >= 3000;
```
- ### 多表查询
```sql
## 查询员工的编号, 姓名, 部门编号, 部门名称, 而部门名称不在emp表而在dept表中, 就要采用多表查询
SELECT * FROM emp;
SELECT * FROM dept;

## 多表查询
## 交叉连接: cross join, cross在Mysql中可以省略, Oracle中不可以
SELECT * FROM emp CROSS JOIN dept; -- 14×4条数据, 为笛卡尔乘积: 无实际意义

## 自然连接: natural join, 自动匹配所有同名列, 并且用名列只展示一次
SELECT * FROM emp NATURAL JOIN dept;

## 缺点: 查询字段时, 没有指定字段所属的数据库表, 效率低, 指定表名即可解决
SELECT emp.empno, emp.ename, emp.sal, dept.dname, dept.loc FROM emp NATURAL JOIN dept;

## 缺点: 表名太长, 给表起别名
SELECT e.empno, e.ename, e.sal, d.dname, d.loc FROM emp e NATURAL JOIN dept d;

## 缺点: NATURAL JOIN会自动匹配所有的同名列, 但是有时候我们希望只匹配部分同名列, 用using子句可解决
SELECT * FROM emp e INNER JOIN dept d USING(deptno); -- inner可以省略, using采用的是内连接

## 缺点: using关联的字段必须是同名的, 可通过内连接on子句解决
SELECT * FROM emp e INNER JOIN dept d ON (e.deptno = e.deptno);

## 多表查询的类型总结:
-- 交叉连接 CROSS JOIN
-- 自然连接 NATURAL JOIN
-- 内连接 USING 子句
-- 内连接 ON 子句, 用的最多

SELECT * FROM emp e INNER JOIN dept d ON (e.deptno = e.deptno) WHERE sal > 3500;

## 条件
-- 筛选条件: WHERE HAVING
-- 连接条件: ON USING NATURAL
-- SQL-1999语法: 筛选条件和连接条件是分开的
```

```sql
## 右外连接: RIGHT OUTER JOIN 表示右边的表就算不匹配也可以查看出效果
SELECT * FROM emp e RIGHT OUTER JOIN dept d ON e.deptno = d.deptno;

SELECT * FROM emp e FULL OUTER JOIN dept d ON e.deptno = d.deptno;

## 并集, 不去重, 效率高
SELECT * FROM emp e LEFT OUTER JOIN dept d ON e.deptno = d.deptno 
UNION ALL 
SELECT * FROM emp e RIGHT OUTER JOIN dept d ON e.deptno = d.deptno;
```

- ### 三表连接查询
```sql
## 查询员工的编号, 姓名, 薪水, 部门编号, 部门名称, 薪水等级
SELECT * FROM emp;
SELECT * FROM dept;
SELECT * FROM salgrade;

SELECT e.ename, e.sal, e.empno, e.deptno, d.dname, s.*
FROM emp e 
RIGHT JOIN dept d 
ON e.deptno = d.deptno
INNER JOIN salgrade s
ON e.sal BETWEEN s.losal AND s.hisal;

## 自连接查询

## 查询员工的编号, 姓名, 上级编号, 上级的姓名
SELECT * FROM emp;

SELECT e1.empno 员工编号, e1.ename 员工姓名, e1.mgr 领导编号, e2.ename 上级姓名
FROM emp e1
INNER JOIN emp e2
ON e1.mgr = e2.empno;

FROM emp e1
LEFT JOIN emp e2
ON e1.mgr = e2.empno;
```

- ### 单行子查询
```sql
## 何为子查询
-- 一条SQL语句中有多个select
## 执行顺序
-- 先执行子查询, 再执行外查询
## 不相关子查询
-- 子查询可以独立运行, 称为不相关子查询
## 不相关子查询分类
-- 根据子查询的结果行数, 可以分为单行子查询和多行子查询

## 引入子查询
-- 查询所有比"clark"工资高的员工的信息
SELECT sal FROM emp WHERE ename = 'clark'; -- 步骤1: "clerk"工资
SELECT * FROM emp WHERE sal > 2450; -- 查询所有比2450高的员工

-- 子查询:
SELECT * FROM emp WHERE sal > (SELECT sal FROM emp WHERE ename = 'clark');

## 单行子查询
-- 查询工资高于平均工资的雇员名字和工资
SELECT ename, sal
FROM emp
WHERE sal > (SELECT AVG(sal) FROM emp); 

-- 查询和clark同一部门且比他工资低的雇员名字和工资
SELECT ename, sal
FROM emp
WHERE deptno = (SELECT deptno FROM emp WHERE ename = 'clark') 
AND sal < (SELECT sal FROM emp WHERE ename = 'clark');

-- 查询职务与scott相同, 比scott雇佣时间早的雇员信息
SELECT *
FROM emp
WHERE job = (SELECT job FROM emp WHERE ename = 'sc0tt')
AND hiredate < (SELECT hiredate FROM emp WHERE ename = 'sc0tt')
```
- ### 多行子查询
```sql
## 多行子查询

-- 查询 [部门20中职务同部门10的雇员一样的] 雇员信息
-- 1.查询雇员信息
SELECT * FROM emp;
-- 2.查询部门20的职位
SELECT job FROM emp WHERE deptno = 20;
-- 3.查询部门10的职位
SELECT job FROM emp WHERE deptno = 10;
-- 4.汇总成多行子查询
SELECT * FROM emp
WHERE deptno = 20
AND job IN (SELECT job FROM emp WHERE deptno = 10);
-- 也可以写成:
SELECT * FROM emp
WHERE deptno = 20
AND job = ANY(SELECT job FROM emp WHERE deptno = 10);


-- 查询工资比所有的"SALESMAN"都高的雇员的编号, 名字和工资
-- 1.查询雇员的编号, 名字和工资
SELECT empno, ename, sal FROM emp;
-- 2.查询"SALESMAN"的工资
SELECT sal FROM emp WHERE job = 'salesman';
-- 3.查询工资比所有的"SALESMAN"都高的雇员的编号, 名字和工资
SELECT empno, ename, sal
FROM emp
WHERE sal > ALL(SELECT sal FROM emp WHERE job = 'salesman');
-- 也可以写成单行子查询:
SELECT empno, ename, sal
FROM emp
WHERE sal > (SELECT MAX(sal) FROM emp WHERE job = 'salesman');


-- 查询工资低于任意一个"CLERK"的工资的雇员信息
-- 1.查询雇员信息
SELECT * FROM emp;
-- 2.查询工资低于任意一个"CLERK"的工资的雇员信息
SELECT *
FROM emp 
WHERE sal < ANY(SELECT sal FROM emp WHERE job = 'clerk')
AND job != 'clerk';
-- 也可以写成:
SELECT *
FROM emp 
WHERE sal < (SELECT MAX(sal) FROM emp WHERE job = 'clerk')
AND job != 'clerk';
```
- ### 相关子查询
```sql
## 区别
-- 不相关子查询: 子查询可以独立运行, 先运行子查询, 再运行外查询
-- 相关子查询: 子查询不可以独立运行, 并且先运行外查询, 再运行子查询


## 查询最高工资的员工 (不相关子查询)
SELECT * FROM emp WHERE sal = (SELECT MAX(sal) FROM emp);


## 查询本部门最高工资的员工 (相关子查询)
-- 1.通过不相关子查询实现:
SELECT * FROM emp WHERE deptno = 10 and sal = (SELECT MAX(sal) FROM emp WHERE deptno = 10)
UNION
SELECT * FROM emp WHERE deptno = 20 and sal = (SELECT MAX(sal) FROM emp WHERE deptno = 20)
UNION
SELECT * FROM emp WHERE deptno = 30 and sal = (SELECT MAX(sal) FROM emp WHERE deptno = 30);
-- 2.通过相关子查询
SELECT * FROM emp e WHERE sal = (SELECT MAX(sal) FROM emp WHERE deptno = e.deptno) ORDER BY deptno


## 查询工资高于其所在岗位的平均工资的那些员工 (相关子查询)
-- 不相关子查询
SELECT * FROM emp WHERE job = 'clerk' AND sal >= (SELECT AVG(sal) FROM emp WHERE job = 'clerk')
UNION
...

-- 相关子查询
SELECT * FROM emp e WHERE sal >= (SELECT AVG(sal) FROM emp WHERE job = e.job);
```
